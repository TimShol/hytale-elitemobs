---
title: Visuals
description: Configure model scaling and nameplate styling.
---

Visual settings are defined in `visuals.yml`.
They control how elites look and how clearly they are identified.

## Model Scaling

```yaml
Model:
  enableModelScaling: true
  modelScaleMultiplierPerTier: [0.74, 0.85, 0.96, 1.07, 1.18]
  modelScaleRandomVariance: 0.04
```

Model scaling affects hitboxes. Keep changes small.
Use scaling to communicate tier strength visually.

## Nameplates

```yaml
Nameplates:
  nameplateMode: "RANKED_ROLE"
  nameplatePrefixPerTier: ["[•]", "[• •]", "[• • •]", "[• • • •]", "[• • • • •]"]
  nameplatesEnabledPerTier: [true, true, true, true, true]
  nameplateMustContainRoles: []
  nameplateMustNotContainRoles: []
```

## Family Prefixes

Tier prefixes can be customized per mob family:
If a family key is missing, `default` is used.

```yaml
Nameplates:
  tierPrefixesByFamily:
    goblin: ["Sneaky", "Cutthroat", "Brutal", "Overseer", "Overlord"]
    zombie: ["Rotting", "Ravenous", "Putrid", "Monstrous", "Evolved"]
    skeleton: ["Broken", "Reforged", "Grim", "Deathbound", "Ascendant"]
```
